# /eureka — control-plane truth is the parent law

## Discovery

The discovery is: **control-plane truth is the parent law**.

The prior step named a real child:

- turn-start surfaces can become **session gates**

An earlier product arc named another:

- **UI boundary truth**

Reading those together makes the parent shape plain. Some surfaces in this project are not mainly content surfaces. They are **control-plane surfaces**: they govern whether work may begin, what state is active, what a boundary changed, and what the receiver is entitled to assume before proceeding.

## The children

Two children are now clean:

### 1. Session gates

On the collaborator side, some turn-start surfaces define the admissible start state of work.

Worked pair:

- `mission-arc` auto-fire
- unread `status: open` entries in `CROSS_AGENT_COMMS.md`

These do not merely improve later work. They govern whether the turn was allowed to begin honestly.

### 2. UI boundary truth

On the user side, some interface surfaces must declare what changed at the boundary itself.

Worked pair:

- scope should be visible before failure
- cross-route persistence must be visible or cleared

These do not merely improve wording. They govern whether the user can truthfully know what is active, available, or still in effect.

## Where middleware fits

Middleware is adjacent, not identical.

Middleware steers downstream composition once admissibility is already settled:

- trajectory middleware
- copy middleware
- interaction middleware
- criterion middleware
- hypothesis middleware

So middleware belongs in the same broad governance neighborhood, but a different slot.

- **middleware** answers: what should the next move be shaped by?
- **control-plane truth** answers: what was active, allowed, or required before the next move could even be honestly made?

That distinction is the discovery.

## Why this matters

Without this parent law, the project keeps rediscovering the same failure in separate languages:

- the collaborator had to infer what pre-work truth was required
- the user had to infer what a boundary changed
- later composition had to compensate for governing state that was never surfaced where it changed

The parent law says the same correction in one sentence:

> governing truth should appear at the layer where permission, state, or sequencing changes, not be reconstructed later from fallout

## Practical test

The practical test is:

> if a collaborator or user has to infer what was active, allowed, or required only after side-effects appear, control-plane truth is under-surfaced

That names the seam more cleanly than either "signaling" or "clarity." It is about the truth of the governing layer.

## Outcome

The project's surface map is now sharper:

- **content/craft surfaces** carry what is said
- **middleware surfaces** steer what gets composed next
- **control-plane surfaces** declare what work/state is valid before or at the moment it changes

That is the broader parent law above session gates and UI boundary truth.
