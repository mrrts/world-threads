# /eureka — prompt-stack control language must stay vertically coherent

The discovery is: **prompt-stack control language must stay vertically coherent**.

This came from a run of small prompt edits that looked cosmetic in isolation:

- fixed-mode closing lines softened from `Almost always, obey` into `Obey by default`
- late-slot fixed-mode seals stopped calling themselves `the commandment of this chat` and started naming the `active length contract`
- the Auto micro-seal stopped saying `Cut ornament before substance` and started saying `Keep the substance; cut ornament first`
- the main Short / Medium / Long opening lines were brought into alignment with that same contract language

Read one by one, those are copy cleanups. Read together, they name a structural failure mode.

## The real failure mode

When one governing rule is repeated across a prompt stack, the first thing to drift is often **not the rule's logic** but its **restated tone**.

The upstream layer gets sharper or truer:

- `Auto` becomes a compass, not a vacuum
- fixed response lengths become active contracts rather than barked commandments
- brevity guidance protects substance rather than sounding like trim-for-trim's-sake

Then a lower slot keeps an older shape:

- commandment rhetoric
- hidden quota language
- rougher trim-order bark
- more mechanical obedience phrasing

The model is then not receiving one law at several reminder points. It is receiving **the same law in two different registers**.

That is the discovery: repeated control language is a **vertical surface**, not a pile of independent strings.

## Why this matters

The project already has a law about register drift:

- register is not mood-performance
- trust is not aura-performance
- Auto mode is a compass, not a vacuum

Vertical prompt coherence is the control-language sibling of those.

It says: a prompt stack can be doctrinally right in its main block and still drift wrong in its reinforcement layers. The late reminder, the fallback string, the micro-seal, the repeated runtime warning — these are where older rhetoric can keep living after the parent doctrine has moved on.

That means small wording passes in these repeated slots are not merely polish. They are repair work on a split control surface.

## The test

The practical test is clean:

> if a lower slot could make the model feel more barked-at, more secretly quota-bound, or more mechanically trimmed than the parent doctrine intends, vertical prompt coherence has split

This is not about flattening all control language into softness.

The length rules are still real.
The contracts are still binding.
The carve-outs are still narrow.

The point is that each layer should teach the **same kind of obedience**.

## Practical consequence

When auditing prompt stacks, don't only ask:

- is the rule present?
- does the late slot repeat it?
- do the substrings still compile-check?

Also ask:

- does the late slot name the same law the same way?
- does the fallback string preserve the same doctrinal truth?
- is the micro-seal reinforcing the parent rule, or smuggling back in an older rhetoric?

If the answer is no, treat the repeated strings as one vertical repair surface.
