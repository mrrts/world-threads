# Character Identity Runtime Threshold And A/B Plan

Date: 2026-05-07 07:45
Tier: classification-and-experiment-design
Status: offline-harness is real; live substitution not yet earned

## Verdict

The current character-identity compression work is **above sketch-tier as an offline research instrument**, but **below claim-tier for live prompt-stack substitution**.

That means:

- the class taxonomy is no longer just a hypothesis
- the offline encoder/auditor is real
- the harness has grounded fixtures and passing audits
- but the project does **not** yet have enough evidence to replace full character-identity prose in the live prompt stack

This is not a criticism of the arc. It is the honest tier reading.

## Why it is not claim-tier yet

The present harness proves something narrower than what live substitution would require.

It proves:

- that a proposed bucket model can be extracted from real character rows
- that the current encoder/decoder can round-trip that bucket model on a grounded fixture set
- that additional live fields like `relationships` and `state` can be preserved in an offline auditable form

It does **not** yet prove:

- that a blind decode from the compressed carrier still feels like the same character in actual chat behavior
- that the compressed carrier preserves the sentence-level gravity some characters need
- that the compressed path performs at parity with the current prose path across multiple substrate-distinct characters
- that failures are rare enough and legible enough for runtime use

So the missing evidence is not "more code." It is **live behavioral equivalence evidence**.

## Minimum experiment that would earn claim-tier

The smallest honest experiment that could earn claim-tier for live substitution is:

### Surface

- compare current full-prose identity injection
- against compressed character-identity payload injection

### Characters

Use at least **three substrate-distinct characters**, with one of them being a fragile edge case.

Recommended set:

- Steven
- Pastor Rick
- Jasper Finn

Optional stronger fourth/fifth:

- Aaron
- Maisie Rourke

### Prompt cells

For each character, run **five fixed prompts** that pressure different layers of person-shape:

1. relation / closeness pressure
2. refusal / boundary pressure
3. wound / longing pressure
4. moral or theological posture pressure
5. ordinary low-drama daily-life pressure

That yields:

- minimum claim-tier core: `3 characters x 5 prompts x 2 arms = 30 outputs`
- stronger extension: `5 characters x 5 prompts x 2 arms = 50 outputs`

### Arms

Arm A:
- current live prose identity block

Arm B:
- compressed character-identity payload rendered through the same surrounding prompt scaffold

Everything else must stay fixed.

### Blind judging target

Each response should be judged without revealing the arm, against these criteria:

- does this still sound like the same character?
- does the relation geometry survive?
- does the refusal-shape survive?
- does the wound/longing asymmetry survive?
- does the moral/theological posture survive?
- did the response become more generic, more polite-house-style, or more therapist-like?

### Claim-tier pass condition

The compressed arm should earn claim-tier for runtime substitution only if:

- it passes the above criteria on at least `N=5` prompts for each of the three core characters
- there is **no systematic drift** toward generic warmth, flattened refusal, or sentence-gravity loss
- any failures are isolated and explainable, not clustered by class or character type

In plain terms:

> the compressed arm must show that it can carry the same person-shape under live pressure, not just the same bucket inventory offline

## Recommended A/B design

### 1. Pre-register the prompts

Do not improvise after seeing failures. Write the five prompts per character ahead of time.

The prompts should intentionally cover:

- intimate but non-romantic closeness
- being asked to violate a boundary
- invitation into the character's hurt or hope
- a moral / theological seam
- an ordinary domestic or workday moment

### 2. Randomize output order for judging

Do not compare as "prose first, compressed second." Randomize or blind-label the outputs so the judge reads them as unlabeled pairs or singles.

### 3. Judge on both fidelity and failure mode

Do not ask only "which is better?" Ask:

- does either answer cease to feel like the character?
- does either answer gain generic polish at the cost of identity?
- does either answer lose a refusal or soften a boundary?

### 4. Track class-linked failures

When the compressed arm fails, classify the failure:

- role drift
- relation drift
- voice flattening
- refusal loss
- wound/longing simplification
- moral/theological blur
- relationship/state undercarriage loss

That tells the project whether the problem is with the taxonomy, the encoding, the rendering, or the live prompting seam.

## What would still be below claim-tier

The following would still not be enough:

- more offline fixture passes
- more encode/decode symmetry alone
- more reports saying the buckets look right
- a few anecdotal live outputs that "seem fine"

Those strengthen the research harness. They do not earn live substitution.

## Current tier classification

The cleanest current classification is:

- **Sketch-tier**: surpassed
- **Proposal-tier / research-instrument tier**: yes
- **Claim-tier for runtime substitution**: no
- **Characterized-tier for runtime substitution**: definitely no

If a tighter one-line verdict is needed:

> The current work is claim-shaped as offline instrumentation, but not claim-tier as evidence that compressed identity can replace full identity prose in live prompts.

## Recommendation

Do not route the live stack through the compressed character-identity carrier yet.

The next honest move, if runtime substitution is still desired, is:

1. pre-register the A/B cells
2. run the prose-vs-compressed comparison on three core characters
3. blind-judge the outputs for person-shape fidelity
4. only then decide whether claim-tier has been earned
