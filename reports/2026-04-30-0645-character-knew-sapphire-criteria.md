# Character-Knew Sapphire criteria (third-crown contract)

This document defines the earning contract for the **third Sapphire-class crown** in the Empiricon architecture.

Target triad:

1. The Cornerstone Inequality (earned)
2. The Receipt of The Empiricon (earned)
3. Character-Knew Sapphire formalization (not yet earned)

This contract exists to prevent vibe-firing, adjacency-promotion, or relabeling of prior crown events.

## Scope

The claim to be earned is narrower than "characters can ever answer well":

**Characters can surface project doctrine in their own idioms under heterogeneous elicitation conditions at maximally-stable convergence, with non-overlap from the two existing Sapphire crowns.**

## Pass conditions (all required)

1. **Separable witness stack**
   - At least 3 independent witness classes with different failure modes.
   - Witnesses cannot be re-used as sole support from Cornerstone/Receipt without fresh discriminating evidence.
2. **Heterogeneous elicitation**
   - Evidence must include at least two elicitation classes (e.g. direct probe, ordinary in-world conversation, meta-craft question, cross-model transfer).
3. **Axis clarity**
   - Each witness labeled by axis (sender-side, receiver-side, strict blind-reader, etc.).
   - Tier assigned per axis; no cross-axis tier laundering.
4. **Non-overlap statement**
   - Written section proving what this crown adds that Cornerstone and Receipt do not already carry.
5. **Canonical synthesis artifact**
   - Single report that includes witness table, failure-mode table, falsifier outcomes, and explicit verdict.
6. **Apparatus-honest refusal history**
   - At least one named dry-well / non-fire decision in the run-up, showing the same discipline that earns also refuses.

## Disqualifiers (any one blocks fire)

- Relabeling Turn 12 Character-Knew as Sapphire without new separable evidence.
- Class inflation by rhetorical summary ("it feels equivalent") without witness/falsifier path.
- Re-using the same substrate class as multiple "independent" witnesses without discrimination.
- Missing strict non-overlap section against Cornerstone + Receipt.
- Missing canonical synthesis artifact.

## Falsifier expectations for this crown

A valid attempt must name and run falsifiers that could actually collapse the claim. Minimum:

1. **Anchor-specificity falsifier** — fails if doctrine-surfacing holds only on one favored character anchor.
2. **Prompt-echo falsifier** — fails if responses merely mirror probe wording rather than structurally reframing.
3. **Cross-surface transfer falsifier** — fails if the claim does not survive at least one substrate/surface transfer test.

## Verdict format (required)

- Verdict: EARN / NOT EARNED
- If EARN: crown noble name + class tag + earning timestamp + artifact path
- If NOT EARNED: closest-failing condition + next discriminating test

## Governance note

This is a contract document, not an argument for immediate fire.  
Until all pass conditions are met and no disqualifier is triggered, Character-Knew Sapphire remains open.
