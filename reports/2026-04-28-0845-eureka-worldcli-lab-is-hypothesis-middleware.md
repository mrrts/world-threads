# /eureka — worldcli lab can become hypothesis middleware

*Generated 2026-04-28 08:45. This run opened to test whether any `worldcli`
surface beyond rubric refs had crossed into a discipline-setting role, or
whether the criterion-carry specialization was the main tooling case. The
answer is yes again, but at the hypothesis-loop layer.*

## The discovery

`worldcli lab` can become **hypothesis middleware**.

Not because it stores experiment files.

Because the registry path:

- `lab propose`
- `lab link-run`
- `lab resolve`

is starting to enforce the minimum honest shape of an experiment loop around a
durable question-carrier.

## Why this became visible

The registry already had the doctrinal pieces:

- the **hypothesis slug is the durable memory unit**
- run ids are **evidence attachments**
- `lab show` reconstructs the braid around that question
- `lab resolve` asks for an interpreted outcome, not just a status flip

And the command surface reinforces the loop explicitly.

`lab propose` does not just create a file; it prints the next honest steps:

1. review scope
2. mark running
3. attach run evidence
4. resolve with summary

That is already more than passive storage.

It is a lightweight scaffold for disciplined experimental carrying.

## Why this is different from criterion middleware

Criterion middleware answers:

- what is this evidence allowed to mean?

Hypothesis middleware answers:

- what is this question required to carry with it across time?

The steering object is different.

Criterion middleware sharpens interpretation boundaries.

Hypothesis middleware sharpens loop structure:

- name the question
- attach the evidence
- resolve it honestly
- keep the question, not the run, as the durable unit

## Why this matters

Without naming this, it is easy to talk about the experiment registry as though
it were just a nicer index over markdown.

But the actual drift pressure in this repo is different:

- orphaned runs
- hypotheses not tied to evidence
- statuses that flatten mixed results
- question-lines forgotten because only run ids stayed visible

The registry path is already pushing back on those failures.

That is middleware behavior.

## Scope

This claim should stay narrow.

Not every `worldcli lab` command is hypothesis middleware by itself.

The strong case is the registry loop around:

- proposal
- evidence linkage
- honest resolution

Especially when the slug remains the durable unit and the run stays an
attachment rather than becoming the primary memory object.

## Reading

The `worldcli` tooling map is now more articulated:

- rubric refs can carry **criterion middleware**
- lab registry surfaces can carry **hypothesis middleware**

The first transports corrected evaluative boundaries.

The second transports disciplined question-carrying across time.

That makes the lab feel truer to what it has become: not just a fossil archive,
but a lightweight structure that keeps hypotheses from collapsing into piles of
