Across many craft-note additions and refinements, Ryan has repeatedly applied the same meta-pattern: a rule that pins a failure mode should almost always leave a sliver of space for the earned counter-instance. Observed in the 2026-04-22 guided craft-notes pass (commit TBD): 9 of the 50 notes reviewed got explicit carve-out requests.

Specific carve-outs he's requested (exemplars, not exhaustive):
- Anti-ribbon closings → but a genuinely earned witty close is fine once in a while (the pattern is the failure, not the instance)
- Softening an observation → flattery if writer-driven, craft if character-driven (whose feeling drives the soften?)
- No dramatic self-awareness → earned moment of articulate clarity is the right move when something has actually clicked
- Anti-grandiosity over ordinary connection → comic-pomposity as a named character trait, AND a character's heart genuinely overflowing
- Awe belongs to the reader not the residents → but caught, specific moments of awe ARE allowed (the fisherman stopping at this sunrise)
- Answer the actual line → genuinely motivated redirect is allowed (with real in-world cause)
- Don't over-narrate apparatus → compelling/vivid lead-up is allowed to breathe
- Don't analyze the user → friends with long history AND rude strangers can step into analyst posture
- Don't stack advice → fourth-wall introspection mode unblocks real recommendations
- Redirect without announcing → head-on facing is equally valid
- Memory ambushes (uncued arrival) → deliberate digging / story-telling is also valid

**Why:** An absolute rule produces rigid, rule-shaped output. The earned-exception pattern produces rule-as-strong-default with honesty about when the default fails. Characters who NEVER break the rule read as mechanical; characters who ALWAYS break it read as ungoverned. The default-plus-earned-exception shape is load-bearing for how this project renders real personhood.

**How to apply:**
- When drafting a new craft note that bans or restricts something, proactively ask: "is there an earned instance where the opposite is the right move?" If yes, name it as an explicit exception with concrete conditions.
- When trimming existing notes, PRESERVE earned-exception clauses unless there's specific data that they're dead. They're doing deliberate work.
- The test I've been using for carve-outs: (1) what's the specific earned case? (2) what condition distinguishes it from the failure mode? Both must be nameable in the prompt.
- Watch for: any rule framed as "NEVER" or "DO NOT" without an exception. Those are candidates for a carve-out pass.
- Do NOT turn every rule into a fuzz of exceptions — the DEFAULT must stay load-bearing. The exception is rare, specific, earned; not a loophole.
