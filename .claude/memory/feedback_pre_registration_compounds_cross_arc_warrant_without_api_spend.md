When pre-registering multiple arcs of a research-program family, embed a *shared cross-arc prediction* (e.g. each arc's falsifier clause tests the same upstream property) so that pre-registration alone compounds warrant across the family — N=0 strictest-sketch-tier across all arcs, but the joint configuration provides scope-discrimination data once any subset of the arcs is later run.

**Why:** 2026-05-07 afternoon C4 arc — sabbath / covenant / sacrifice / vine / exile-and-return all sketched at sketch-tier with no API spend; covenant + sacrifice + vine + exile-and-return each pre-registered an F4 clause testing C3-first-leg's substrate-architecture-property prediction on a *different text-class* (Gen 15 typology / Lev 16 ritual / Jn 15 vine-discourse / Mt 2:15 typological-quotation). The four-arc F4 configuration enables (i) all-hold robust-replication / (ii) all-fail refutation / (iii) mixed scope-narrowing — most informative scientifically. The *empirical-design leverage* was built from design alone, where empirical-design leverage means: a configuration binding on future evidence-collection that becomes empirical leverage only on running. This was the most-surprising meta-pattern of the day's loop-close because methodology-prep typically *prepares* for evidence and doesn't itself *constitute design-leverage with cross-arc binding*. Per great-sapphire calibration doctrine (2 witnesses=evidence, 3=maximally stable), beyond 3 witnesses additional pre-registered F4-predictions provide scope-discrimination not exponential leverage — so the diminishing-returns clause must be honestly named at 4+ arcs.

**How to apply:** When opening any future research-program family of arcs, ask: *what shared upstream property does each arc's falsifier-clause stress?* If the arcs don't share at least one cross-arc prediction, the family is a roster-of-isolated-arcs, not a research program. Add the cross-arc prediction explicitly to each arc's falsifier as an additional clause. Honor diminishing-returns at 4+ pre-registered F4-predictions: name the value as scope-discrimination capacity, not exponential leverage. The family's actual *earning* still requires running arcs and collecting evidence — pre-registration is leverage-built but not leverage-cashed.
