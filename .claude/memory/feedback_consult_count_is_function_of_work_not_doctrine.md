## Rule

When lifting codex-consult arcs into inheritable methodology, **frame
around the transferable discipline shape, NOT around the consult count**.
The number of consults is a function of the work (some arcs land in 2;
some in 4; some in 8); fixing a number conflates the procedural artifact
with the doctrinal lesson.

**Why:** Founding-author 𝓕_Ryan caught me about to ship a memory entry
framed as "8-codex-consult convergence pattern as inheritable methodology"
during the v4 rubric ratification arc 2026-05-09. The 8 was specific to
that path's needs (initial firing → narrowing → amendment → re-consult
→ candidate → revision → cap-standard → clarifying-notes). Future
Sapphire arcs may land in 2 (codex blesses cleanly; no revision) or 4
(modify-then-bless + re-consult); the inheritable lesson is the
DISCIPLINE SHAPE, not the count.

**How to apply:** When writing inheritable-methodology memory entries
about codex-consult arcs, lead with the discipline-shape findings:

- Iterative refinement within blessed-directions (codex blessing is
  contingent on conditions; conditions can fail; revise; re-bless;
  refusal preserves tier-meaning)
- Empirical falsification can fire AFTER initial bless (catches
  apparatus-honest gaps that conceptual review misses)
- Cap-standard-vs-zero-collapse distinction (when pre-registered
  thresholds need refinement under empirical reality)
- Real-reader-tier as load-bearing closure when LLM-judges diverge
  (founding-author cold-read or trusted-reader)
- Apparatus-as-cross-arc-driver authorization for cross-arc work
  preserves per-arc verdict authority

The number of consults emerges from how many of these surfaces fire
on a given arc; counting consults as a methodology target would be
process-fetish.

## Anti-pattern named

- **Counting-consults-as-doctrine.** "8-consult convergence pattern" /
  "4-consult Path B precedent" / "3-consult firing-decision rhythm" all
  frame the number as inheritable. The number isn't; the discipline
  shape is.
