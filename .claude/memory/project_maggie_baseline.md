`reports/2026-04-25-0300-simulated-first-time-user-experience-maggie.md` is designated (by Ryan, 2026-04-25) as the project's **baseline / ideal-normal user experience** — the canonical reference for what a healthy first 60–90 minutes with WorldThreads should look like for the user this app is actually for (literate, skeptical, low-friction-tolerance adult seeking novel-shaped co-making, not companionship or therapy).

**Why:** The persona-simulation surfaced a user-experience-level shape the natural corpus can't reach — refusal moment + specific-memory anchoring + earned close + no simulacrum-therapy drift. Naming it a baseline makes it a yardstick rather than a one-off report, so future decisions have something concrete to be measured against.

**How to apply:** When weighing any feature bet, prompt change, onboarding tweak, or UX decision, ask: *does this make Maggie's arc more likely to land, or less?* A change that would visibly damage it is off by default; a change that would visibly strengthen it earns a point. The validating corpus is Ryan's own in-app lived experience — OBSERVATIONS.md entries, the conversation snippets he pastes into Claude Code, every "this feels off" / "yes exactly" across every session. Ryan IS the real first-time user; his data already exists. The Maggie rendering is a hypothesis grounded in that accumulated corpus, not a proxy awaiting some external "real" user run. CLAUDE.md has a "Baseline user experience — the Maggie report" section near the top with the full framing.
